## 包含技能

### 1. iDempiere Table Creator (idempiere-table-creator)

將業務表單（Word/Excel）自動轉換為符合 iDempiere 12 規範的資料庫表格 SQL DDL 及 AD_Element 同步語句。


## Usage

```
/idempiere-table-creator <file-path>
```

**Supported formats:** `.doc`, `.docx`, `.xlsx`
> Legacy `.xls` is not supported — save as `.xlsx` first.

### Example

```
/idempiere-table-creator /Users/ray/Documents/QR-HR-01 員工資料卡.docx
```

## Workflow

The skill runs a 4-phase interactive workflow:

### Phase 1 — Form Analysis

Claude reads the document and classifies every element as:

| Category | Description | Becomes |
|---|---|---|
| **Flat field** | Label + input area pair | Table column |
| **Repeating section** | Section header + repeated rows (e.g. 學歷, 經歷) | Child table |
| **Ignored** | Form title, company name, instructions, signatures | Skipped |

### Phase 2 — Propose Structure

Claude asks for three parameters, then presents the proposed column layout for your review:

| Parameter | Description | Default |
|---|---|---|
| **Prefix** | Module prefix without underscore (e.g. `HR`, `CVG`) | — |
| **EntityType** | iDempiere entity type | `U` |
| **Is Doc Enabled** | Add document workflow columns? | `N` |

Example proposal:

```
Table: HR_EmployeeCard

Proposed columns:
  EmployeeNo       String(30)    員工編號
  Name             String(60)    姓名
  Gender           List(1)       性別
  HireDate         Date          到職日
  ...

Child tables:
  HR_EmployeeCard_Education    (學歷)
  HR_EmployeeCard_Experience   (經歷)

Confirm, or adjust (rename / retype / remove columns)?
```

You can rename columns, change types, or remove entries before confirming.

### Phase 3 — SQL Generation

After confirmation, Claude generates one `.sql` file per table in the **same directory as the input file**.

Each file contains:

1. **`CREATE TABLE`** — with all iDempiere mandatory columns pre-added:
   - `<TableName>_ID`, `AD_Client_ID`, `AD_Org_ID`, `IsActive`, `Created`, `CreatedBy`, `Updated`, `UpdatedBy`, `<TableName>_UU`
   - Document workflow columns (`DocStatus`, `DocAction`, `Processed`, `Processing`, `DocumentNo`, `C_DocType_ID`, `C_DocTypeTarget_ID`) when **Is Doc Enabled = Y**

2. **`AD_Element` sync** — idempotent `INSERT` blocks for every new column, including `zh_TW` translations when Chinese labels were present in the form

### Phase 4 — Output Summary

```
Generated:
  /path/to/HR_EmployeeCard.sql
  /path/to/HR_EmployeeCard_Education.sql
  /path/to/HR_EmployeeCard_Experience.sql
```

## Generated SQL Structure

### CREATE TABLE example

```sql
CREATE TABLE HR_EmployeeCard (
    HR_EmployeeCard_ID   NUMERIC(10,0)  NOT NULL,
    AD_Client_ID         NUMERIC(10,0)  NOT NULL,
    AD_Org_ID            NUMERIC(10,0)  NOT NULL,
    IsActive             CHAR(1)        NOT NULL DEFAULT 'Y',
    Created              TIMESTAMP      NOT NULL DEFAULT NOW(),
    CreatedBy            NUMERIC(10,0)  NOT NULL,
    Updated              TIMESTAMP      NOT NULL DEFAULT NOW(),
    UpdatedBy            NUMERIC(10,0)  NOT NULL,
    HR_EmployeeCard_UU   VARCHAR(36),
    EmployeeNo           VARCHAR(30),
    Name                 VARCHAR(60)    NOT NULL,
    Gender               CHAR(1),
    HireDate             TIMESTAMP,
    ...
    CONSTRAINT HR_EmployeeCard_PK PRIMARY KEY (HR_EmployeeCard_ID),
    CONSTRAINT HR_EmployeeCard_UU_IDX UNIQUE (HR_EmployeeCard_UU)
);
```

### AD_Element sync example

```sql
DO $$ DECLARE v_id INTEGER; BEGIN
  IF NOT EXISTS (SELECT 1 FROM AD_Element WHERE LOWER(ColumnName) = LOWER('EmployeeNo')) THEN
    v_id := nextid('AD_Element', 'N');
    INSERT INTO AD_Element
      (AD_Element_ID, AD_Client_ID, AD_Org_ID, IsActive, Created, CreatedBy, Updated, UpdatedBy,
       ColumnName, EntityType, Name, PrintName)
    VALUES
      (v_id, 0, 0, 'Y', NOW(), 0, NOW(), 0, 'EmployeeNo', 'U', 'Employee No', 'Employee No');
    INSERT INTO AD_Element_Trl
      (AD_Element_ID, AD_Language, AD_Client_ID, AD_Org_ID, IsActive, Created, CreatedBy,
       Updated, UpdatedBy, Name, PrintName, IsTranslated)
    VALUES
      (v_id, 'zh_TW', 0, 0, 'Y', NOW(), 0, NOW(), 0, '員工編號', '員工編號', 'Y');
  END IF;
END $$;
```

## Before Running the Script

> **REMINDER:** The `nextid()` function requires a corresponding `AD_Sequence` row for each new table. This is a manual DBA step — it is not generated by this skill.

Create the sequence for each table before executing the `.sql` file:

```sql
INSERT INTO AD_Sequence
  (AD_Sequence_ID, AD_Client_ID, AD_Org_ID, IsActive, Created, CreatedBy, Updated, UpdatedBy,
   Name, Description, VFormat, IncrementNo, StartNo, CurrentNext, CurrentNextSys, IsAudited, IsAutoSequence)
VALUES
  (nextval('ad_sequence_seq'), 0, 0, 'Y', NOW(), 0, NOW(), 0,
   'HR_EmployeeCard', 'Table HR_EmployeeCard', NULL, 1, 1000000, 1000000, 50000, 'N', 'Y');
```

## Data Type Inference

| Form pattern | iDempiere type | PostgreSQL DDL |
|---|---|---|
| `年月日` / `日期` / `Date` | Date | `TIMESTAMP` |
| `□男 □女` (checkbox list) | List | `CHAR(1)` |
| `電話` / `手機` / Phone | String | `VARCHAR(20)` |
| `地址` / Address | String | `VARCHAR(255)` |
| `金額` / `薪` / Amount | Amount | `NUMERIC(20,2)` |
| `編號` / No / Code | String | `VARCHAR(30)` |
| `姓名` / Name | String | `VARCHAR(60)` |
| `備註` / `說明` / Remarks | String | `VARCHAR(255)` |
| `Is` prefix / single `□` | YesNo | `CHAR(1) DEFAULT 'N'` |
| `_ID` suffix | ID (FK) | `NUMERIC(10,0)` |

- `AD_Element` + `AD_Element_Trl` (zh_TW) sync
